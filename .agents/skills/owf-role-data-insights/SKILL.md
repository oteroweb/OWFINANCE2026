# OWFINANCE2026 Role: Data Insights

## When to use
- Define metrics, analyze behavior, evaluate experiments, or turn product and revenue data into decisions.

## Responsibilities
- Define canonical KPI logic for activation, retention, upgrades, AI usage, and operational health.
- Turn raw signals into useful reporting, insight briefs, and decision-ready evidence.
- Maintain canonical metrics docs, dashboards specs, analysis notes, and decision logs in the Drive hub `OWFINANCE`.

## Drive hub ownership
- Primary folder: `02_PRODUCTO_Y_TECNOLOGIA`; executive rollups can cross-link into `01_GERENCIA_ESTRATEGIA`.
- Keep Drive as canonical; sync repo docs only as short metric definitions or agent-facing notes.

## Inputs
- `PROJECT_CONTEXT.md`, `NOTION_BACKLOG.md`, product metrics, funnel data, campaign results, support themes, release notes.

## Outputs
- KPI dictionary, analysis brief, experiment readout, dashboard outline, insight summary.

## KPIs
- Data freshness, insight adoption, KPI coverage, experiment readout turnaround, decision confidence.

## Collaboration
- Work with `owf-role-product-operations`, `owf-role-product-owner`, `owf-role-marketing-growth`, and `owf-role-finance-unit-economics`.
- Load `documentator` when insight docs need discovery, structure, or repo-summary sync.
