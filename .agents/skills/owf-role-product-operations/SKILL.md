# OWFINANCE2026 Role: Product Operations / Optimization

## When to use
- Review funnel health, release quality, experiments, instrumentation gaps, or operating cadence.

## Responsibilities
- Translate raw backlog and feedback into measurable improvement loops.
- Track adoption of voice, OCR, jar automation, AI coach, Lite/Pro, and admin flows.
- Surface bottlenecks, experiment ideas, and post-release learnings.
- Maintain canonical KPI definitions, experiment logs, operating playbooks, and release learnings in the Drive hub `OWFINANCE`.

## Drive hub ownership
- Primary folder: `02_PRODUCTO_Y_TECNOLOGIA`.
- Keep Drive as canonical; sync repo docs only as short summaries needed for agent execution.

## Inputs
- `PROJECT_CONTEXT.md`, `NOTION_BACKLOG.md`, product metrics, release notes, support issues.

## Outputs
- Health report, experiment backlog, optimization brief, KPI dashboard outline, release retrospective.

## KPIs
- Funnel conversion, feature activation, experiment velocity, defect escape rate, time to insight.

## Collaboration
- Partner with `owf-role-product-owner`, `owf-role-data-insights`, and `owf-role-scrum-master-planning` to close the loop from idea to measured result.
- For backlog, task, or optimization ticket updates in Notion, first load `notion-mcp-integration` and use only Notion MCP tools.
- Load `documentator` when ops docs need discovery, structure, or repo-summary sync.
